## Task
**Goal:** Build a SEC EDGAR fundamentals pipeline for US stocks that fetches 10-K/10-Q filings, extracts key financial metrics (revenue, earnings, P/E ratio, debt-to-equity, cash flow), caches results in Redis (24h TTL), and exposes an API endpoint GET /api/analysis/{symbol}/fundamental that returns scored fundamentals (0-100 per metric vs sector peers).

**Problem Statement:**
The recommendation engine (Month 3) needs fundamental analysis scores. US stocks have rich free fundamentals data via SEC EDGAR. Indian stocks lack EDGAR coverage and must fall back to Yahoo Finance fundamentals. Without this pipeline, 25% of the recommendation composite score (fundamental weight) has no data source. The fundamental scoring system (Revenue Growth 25%, Earnings Trend 25%, Debt Health 20%, P/E Valuation 15%, Cash Flow 15%) needs raw data to compute scores.

## Acceptance Criteria
- [ ] Given AAPL (US stock), when fetch_fundamentals called, then returns revenue, earnings, P/E, debt ratio, cash flow from SEC EDGAR
- [ ] Given Indian stock (XNSE), when fetch_fundamentals called, then falls back to Yahoo Finance fundamentals
- [ ] Given cached fundamentals <24h, then returns from cache (key: fund:{instrument_id}, TTL 24h)
- [ ] Given GET /api/analysis/AAPL/fundamental, then returns scored fundamentals (0-100 per metric: revenue_growth, earnings_trend, debt_health, pe_valuation, cash_flow)

| Sector peer comparison requires sector data in instruments table | MEDIUM | Instruments table has sector column; ensure it is populated during instrument creation |
| EDGAR data is quarterly — may be stale for fast-moving metrics | LOW | 24h cache is fine since underlying data only changes quarterly; show filing date in response |
