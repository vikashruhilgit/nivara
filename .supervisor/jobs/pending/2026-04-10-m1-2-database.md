## Task
**Goal:** Set up Alembic with async template and create the initial migration containing all 14 tables from TechSpec v1.3 section 3, with immutable audit_log enforced by DB trigger and role-based access, plus seed data for top 50 NSE + S&P 500 instruments.

**Problem Statement:**
The backend needs a complete database schema before any feature (auth, brokers, portfolio sync) can be built. Without the schema, no other backend job can write or read persistent data. This is a foundational dependency for Jobs 4-8.

## Acceptance Criteria
- [ ] Given Docker running, when `alembic upgrade head` runs, then all 14 tables created: users, instruments, symbol_mappings, broker_connections, device_tokens, positions, orders, fx_rates, corporate_actions, price_history (partitioned by month), recommendations, notifications, calendar_overrides, audit_log (immutable via trigger)
- [ ] Given investiq_app role, when `UPDATE audit_log SET event_type='x'`, then trigger raises exception
- [ ] Given seed script, when `python manage.py seed` runs, then instruments + symbol_mappings populated with top 50 NSE + S&P 500 instruments
- [ ] Given price_history table, when queried, then partitioned by month with composite PK (instrument_id, timestamp)
- [ ] Given DB roles, when investiq_app attempts DELETE on audit_log, then permission denied (REVOKE enforced)
