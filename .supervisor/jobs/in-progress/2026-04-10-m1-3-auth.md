## Task
**Goal:** Implement FastAPI auth endpoints (register, login, refresh, logout) as sole auth authority with JWT RS256 15min access tokens and opaque single-use refresh tokens in Redis, using bearer token pattern (mobile-first, not cookies), argon2 password hashing, and refresh token invalidation on password change.

**Problem Statement:**
The platform has no authentication. All endpoints are publicly accessible. The mobile app needs bearer-based auth (not cookies) with secure token rotation. Auth is a prerequisite for the mobile shell (Job 7) and portfolio sync (Job 8).

## Acceptance Criteria
- [ ] Given valid credentials, when POST /api/auth/register, then returns access_token in JSON body + refresh_token in JSON body (bearer pattern, not cookies)
- [ ] Given valid credentials, when POST /api/auth/login, then returns access_token + refresh_token in JSON body
- [ ] Given valid refresh_token, when POST /api/auth/refresh, then returns new access_token + new refresh_token, old refresh_token invalidated in Redis
- [ ] Given logged out user, when using old refresh_token, then 401 Unauthorized
- [ ] Given protected endpoint, when no/expired access_token in Authorization header, then 401 Unauthorized
- [ ] Given password change, when any existing refresh_token used, then 401 (all tokens invalidated)
- [ ] Given JWT access_token, then payload contains user_id, tier, base_currency, kid (key ID for RS256)
